## 🚀 Deployment Steps

### 1. Prepare Repository

```bash
# Make sure all files are committed
git add .
git commit -m "Ready for deployment"
git push origin main
```

### 2. Update render.yaml

Your `render.yaml` should look like this:

```yaml
services:
  - type: web
    name: crm-central-command
    env: python
    region: oregon
    plan: starter
    buildCommand: |
      pip install -r backend/requirements.txt
      cd frontend && npm install && npm run build
    startCommand: |
      cd backend && gunicorn --worker-class eventlet -w 1 --bind 0.0.0.0:$PORT app:app
    envVars:
      - key: SUPABASE_URL
        sync: false
      - key: SUPABASE_SERVICE_KEY
        sync: false
      - key: SESSION_SECRET
        generateValue: true
      - key: ENVIRONMENT
        value: production
      - key: ALLOWED_ORIGINS
        value: "*"
    healthCheckPath: /api/health
    autoDeploy: true
```

### 3. Deploy to Render

#### Option A: Blueprint (Recommended)
1. Go to [render.com](https://render.com)
2. Click **New** → **Blueprint**
3. Connect your GitHub repo
4. Render will detect `render.yaml`
5. Set environment variables:
   - `SUPABASE_URL`: Your Supabase project URL
   - `SUPABASE_SERVICE_KEY`: Your service_role key
6. Click **Apply**

#### Option B: Manual Setup
1. Click **New** → **Web Service**
2. Connect GitHub repo
3. Configure:
   - **Name**: `crm-central-command`
   - **Environment**: Python 3
   - **Build Command**: 
     ```bash
     pip install -r backend/requirements.txt && cd frontend && npm install && npm run build
     ```
   - **Start Command**: 
     ```bash
     cd backend && gunicorn --worker-class eventlet -w 1 --bind 0.0.0.0:$PORT app:app
     ```
4. Add environment variables
5. Click **Create Web Service**

### 4. Monitor Deployment

Watch the logs for:
```
✓ Installing Python dependencies
✓ Installing Node dependencies  
✓ Building React app
✓ Starting Gunicorn
✓ Flask-SocketIO initialized
✓ Server running on port 10000
```

### 5. Verify Deployment

Visit your Render URL:
- `https://your-app.onrender.com` - Should show React app
- `https://your-app.onrender.com/api/health` - Should return JSON

## 🔧 Production Configuration

### Gunicorn Settings

For production, use **1 worker** with eventlet:
```bash
gunicorn --worker-class eventlet -w 1 --bind 0.0.0.0:$PORT app:app
```

**Why 1 worker?**
- WebSocket connections require sticky sessions
- Flask-SocketIO works best with single worker
- Eventlet handles concurrency

### Scaling Options

If you need more capacity:
1. **Vertical Scaling**: Upgrade Render plan
2. **Redis Adapter**: Add Redis for multi-worker support
   ```python
   socketio = SocketIO(app, message_queue='redis://localhost:6379')
   ```

## 🐛 Troubleshooting

### Build Fails

**Issue**: Node/npm not found
**Solution**: Render provides Node.js automatically

**Issue**: Python packages fail to install
**Solution**: Check `requirements.txt` for typos

### Server Crashes

**Issue**: eventlet import error
**Solution**: Verify `eventlet` in requirements.txt

**Issue**: Port binding error
**Solution**: Use `$PORT` environment variable

### WebSocket Issues

**Issue**: Socket.IO not connecting
**Solution**: Check CORS and allowed origins

**Issue**: Comments not real-time
**Solution**: Verify Flask-SocketIO is running

### Static Files 404

**Issue**: React app not loading
**Solution**: 
1. Check frontend build completed
2. Verify Flask serves from `frontend/build`
3. Check app.py static_folder path

## 📊 Performance Optimization

### 1. Enable Compression
```python
from flask_compress import Compress
Compress(app)
```

### 2. Add Caching
```python
from flask_caching import Cache
cache = Cache(app, config={'CACHE_TYPE': 'simple'})
```

### 3. Optimize React Build
```bash
# In frontend/vite.config.js
build: {
  rollupOptions: {
    output: {
      manualChunks: {
        'react-vendor': ['react', 'react-dom'],
        'video-vendor': ['react-player'],
      }
    }
  }
}
```

## 🔄 CI/CD Pipeline

### GitHub Actions

Create `.github/workflows/deploy.yml`:
```yaml
name: Deploy to Render

on:
  push:
    branches: [main]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-python@v4
        with:
          python-version: '3.11'
      - run: pip install -r backend/requirements.txt
      - run: python -m pytest backend/tests

  deploy:
    needs: test
    runs-on: ubuntu-latest
    steps:
      - run: echo "Render auto-deploys from GitHub"
```

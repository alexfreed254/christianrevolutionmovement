# Deployment Guide - Christ Revolution Movement

This guide covers deploying the CRM platform to **Render** and publishing to **GitHub**.

---

## 📋 Prerequisites

1. **GitHub Account** - for code repository
2. **Supabase Account** - for database and authentication
3. **Render Account** - for hosting
4. **Git installed** on your local machine

---

## 🔧 Step 1: Setup Supabase

1. Go to [supabase.com](https://supabase.com) and create a new project
2. Wait for the project to initialize (2-3 minutes)
3. Go to **Settings → API** and copy:
   - `Project URL` (e.g., `https://xxxxx.supabase.co`)
   - `service_role` key (anon key won't work for admin operations)
4. Go to **SQL Editor** and run the entire `database/schema.sql` file
5. Verify tables were created in **Table Editor**

---

## 🚀 Step 2: Deploy to GitHub

### Initialize Git Repository (if not already done)

```bash
cd christ-revolution-movement
git init
git add .
git commit -m "Initial commit - CRM Central Command"
```

### Create GitHub Repository

1. Go to [github.com](https://github.com) and click **New Repository**
2. Name it: `christ-revolution-movement`
3. Make it **Private** (recommended for production)
4. **Do NOT** initialize with README, .gitignore, or license
5. Copy the repository URL

### Push to GitHub

```bash
# Add GitHub as remote
git remote add origin https://github.com/YOUR_USERNAME/christ-revolution-movement.git

# Push code
git branch -M main
git push -u origin main
```

---

## ☁️ Step 3: Deploy to Render

### Method 1: Using Blueprint (render.yaml) - Recommended

1. Go to [render.com](https://render.com) and sign in
2. Click **New +** → **Blueprint**
3. Connect your GitHub account if not already connected
4. Select the `christ-revolution-movement` repository
5. Render will automatically detect `render.yaml`
6. Set the following environment variables:
   - `SUPABASE_URL`: Your Supabase project URL
   - `SUPABASE_SERVICE_KEY`: Your service_role key from Supabase
   - `SESSION_SECRET`: Will be auto-generated
7. Click **Apply**
8. Wait 3-5 minutes for deployment

### Method 2: Manual Web Service

1. Go to [render.com](https://render.com) and sign in
2. Click **New +** → **Web Service**
3. Connect your GitHub repository
4. Configure:
   - **Name**: `crm-central-command`
   - **Region**: Choose closest to your users
   - **Branch**: `main`
   - **Root Directory**: Leave blank
   - **Runtime**: `Python 3`
   - **Build Command**: `pip install -r backend/requirements.txt`
   - **Start Command**: `cd backend && uvicorn main:app --host 0.0.0.0 --port $PORT --workers 2`
5. Add Environment Variables:
   - `SUPABASE_URL`: Your Supabase URL
   - `SUPABASE_SERVICE_KEY`: Your service_role key
   - `SESSION_SECRET`: Generate a random 32-character string
   - `ENVIRONMENT`: `production`
6. Click **Create Web Service**

---

## 🔐 Step 4: Configure Environment Variables

### On Render Dashboard:

1. Go to your service → **Environment**
2. Add/verify these variables:
   ```
   SUPABASE_URL=https://xxxxx.supabase.co
   SUPABASE_SERVICE_KEY=eyJhbGc...
   SESSION_SECRET=<auto-generated or your secret>
   ENVIRONMENT=production
   ALLOWED_ORIGINS=https://your-app.onrender.com
   ```
3. Click **Save Changes**
4. Service will auto-redeploy

## 🐛 Troubleshooting

### Build Failed

**Check Render Logs:**
1. Go to your service → **Logs**
2. Look for error messages

**Common issues:**
- Missing dependencies → Check `requirements.txt`
- Python version mismatch → Verify `runtime.txt`
- Path issues → Check directory structure

### Application Crashes

**Check Environment Variables:**
```bash
# In Render dashboard → Environment
# Verify all required variables are set
```

**Check Application Logs:**
1. Service → **Logs** → **Application Logs**
2. Look for Python errors or missing configs

### Database Connection Issues

**Verify Supabase credentials:**
- Check `SUPABASE_URL` format (should include https://)
- Verify `SUPABASE_SERVICE_KEY` (should be service_role, not anon key)
- Test connection from local machine first

**Check Supabase project status:**
- Ensure project is not paused (free tier auto-pauses after inactivity)

### Static Files Not Loading

**Check file paths:**
- Ensure `frontend/static` directory exists
- Verify CSS/JS files are in correct subdirectories
- Check browser console for 404 errors

## 🔒 Security Checklist

Before going live:

- [ ] Remove real credentials from `.env.example`
- [ ] Set `ALLOWED_ORIGINS` to your actual domain (not `*`)
- [ ] Enable Supabase Row Level Security (RLS)
- [ ] Use HTTPS only (Render provides this automatically)
- [ ] Set strong `SESSION_SECRET`
- [ ] Review Supabase auth policies
- [ ] Enable Render's DDoS protection
- [ ] Set up database backups in Supabase
- [ ] Configure Sentry or logging for error tracking
